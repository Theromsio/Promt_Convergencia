La convergencia hacia UiPath es ALTAMENTE VIABLE (100% Cobertura). Reemplaza íntegramente las capacidades de Blue Prism eliminando deuda técnica on-premise. DIVERGENCIA (UiPath):
UiPath carece de un modelo nativo de objetos orientados estrictamente a la inmutabilidad transaccional bancaria tan rígida como Blue Prism (VBOs). Su aproximación requiere una estructura de "Try/Catch" ligeramente más manual en secuencias grandes. Además, factores muy específicos de arquitecturas Mainframe On-Premise legadas de misión crítica son estadísticamente más robustos nativamente en el runtime local cerrado de Blue Prism.

5. Análisis Competitivo

Ventajas de la Convergencia a UiPath
  • Hiper-Automatización Efectiva Mientras BP nació para automatización oculta back-office, UiPath es número 1 indiscutido en atender la "última milla" conectando IA (Document Understanding / Copilot) directo a asistentes front de usuarios mortales.
  • Ecosistema Colosal El marketplace de paquetes C# y Python que se inyectan en UiPath es el más grande de la industria. Romper SAP GUI o interactuar con APIs de SFDC tomará segundos versus semanas en construcciones VBO de BP.
  • Fin del On-Prem pesado Migrar a Automation Cloud significa dejar de patchar SO's para BP Database, BP Application Server e interactivas máquinas. Se agiliza la operatividad.
Desafíos y Factores Limitantes
  • Refactor Masivo de Código XML de BP no es traductible 1:1 directo y mágicamente a un workflow XAML de UiPath. Pese a conversores semi-IA de hoy, siempre se requerirá QA manual intensivo y reprogramar lógicas.
  • Caos de Licenciamiento UiPath UiPath es conocido por ofrecer múltiples licencias SKUs complejos (AI units, Document Pages, Unattended versus AI Trust). Puede ser difícil unificar un presupuesto cerrado.

7. Análisis Financiero Consolidado

Contexto: Mantener la plataforma Blue Prism exige costos indirectos muy altos para un departamento TI (Mantenimiento de granjas de Windows Server, DBA para SQL, VPNs) en modelos Unattended, sin dar pie a la democratización "Citizen Developer". Migrar a UiPath no reduce siempre el pago del Opex al vendor RPA, pero recorta gastos periféricos (On-prem DB, On-Prem Security patches). UiPath trae módulos IA incluidos en la arquitectura base evitando contratar APIs de OCR externas (Google Cloud Document AI).

9. Diagrama de Instalación

1

Paso 1: Aprovisionamiento Cloud

Alta del Tenant corporativo centralizado en UiPath Automation Cloud y sincronización directa de identidades mediante Azure / Microsoft Entra ID.

2

Paso 2: Instalación de Desarrollo

Los creadores técnicos descargan UiPath Studio en sus ordenadores locales para comenzar la construcción y refactorización de lógicas heredadas de Blue Prism.

3

Paso 3: Despliegue de Agentes Ejecutores

Instalación silenciosa de UiPath Robot en el footprint de PCs/Servidores del negocio para atender automáticamente la ejecución conectada a las credenciales seguras SaaS.

10. Recomendaciones de Implementación

Fases Rollout de Sustitución Hacia UiPath
  • Fase 1: Mapeo de Arquitectura BP. (Mes 1): Generar PDD (Docs) vía Task Mining para no depender sólo de leer código VBO viejo.
  • Fase 2: UiPath Pilot + AI (Mes 2-4): Migrar un cluster complejo de ERP SAP y acoplar herramientas de Document Understanding para demostrar que UiPath hace el OCR mejor/más rápido que el setup clásico que tenía el negocio.
  • Fase 3: Transición (Mes 5-11): Levantar Automation Cloud, migrar Active Directories a Entra ID Connect e iterar portando .bprelease files a workflows complejos REF. Reducción paulatina de granjas BP Windows.
